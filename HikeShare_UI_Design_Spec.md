# TrailNote 移动端 UI 设计规范

## 1. 项目概述

### 1.1 产品定位

TrailNote 是一款面向户外徒步用户的路线记录与分享工具。用户既可以作为分享者记录徒步轨迹、添加关键点位、上传图像与视频，也可以作为浏览者搜索、查看、收藏和参考他人发布的路线内容。

产品目标：

- 降低徒步路线分享门槛，让路线记录过程足够轻量
- 让浏览者快速判断路线是否值得去、是否适合自己
- 让路线内容除了“轨迹”之外，还具备“关键节点信息”和“现场感”
- 建立可靠、安全、可复用的户外内容社区体验

### 1.2 设计原则

- 一套品牌体验：iOS 与 Android 共用同一套视觉语言、信息架构和组件逻辑
- 非 Material 默认风格：避免 Google 默认 MD3 外观，采用更偏自然探索感的户外品牌体系
- 内容优先：路线、地图、节点信息和媒体内容优先于装饰性 UI
- 户外场景友好：强对比、可单手操作、弱网和强光场景下也易读
- 分享过程低打断：记录轨迹时，添加点位、拍照、录像都应尽量少步骤
- 安全信任优先：重要风险、难度、耗时、补给等信息应醒目

### 1.3 目标用户

1. 记录分享者
   习惯徒步、愿意沉淀路线内容，希望记录过程不复杂。
2. 搜索浏览者
   在出发前查路线信息，希望快速了解风景、难度、路况和真实体验。
3. 轻度内容消费者
   以“看图看视频看推荐”为主，未来可能转化为路线贡献者。

## 2. 产品范围

### 2.1 核心功能

- 登录、注册、找回密码
- 首页内容推荐
- 搜索路线、地点、作者
- 路线详情查看
- 地图查看轨迹和关键点位
- 收藏、点赞、评论、分享
- 开始记录徒步路线
- 记录过程中添加关键点位
- 点位拍照、录视频、填写说明
- 路线编辑与发布
- 个人主页与内容管理

### 2.2 建议补充功能

- 路线草稿自动保存
- 离线地图缓存提示
- 点位标签体系，如“观景点 / 补给点 / 危险路段 / 岔路口 / 营地”
- 路线难度、海拔爬升、路况、适合季节
- 举报与内容审核入口
- 关注作者

## 4. 品牌与视觉方向

### 4.1 品牌气质

关键词：

- 自然
- 探索
- 可靠
- 克制
- 真实

避免：

- 过度科技感霓虹风
- 过度拟物
- Google Material 原生组件既视感
- 过圆、过软、过糖果色

### 4.2 视觉主题

采用“地形图 + 户外装备”混合风格。

设计表达：

- 背景中使用极淡的地形线纹理
- 主色来自森林、岩石、湖泊、日照
- 卡片与地图容器更偏工具感，而不是社交娱乐感
- 通过大图、地图切片、数据标签建立专业可信度

### 4.3 色彩系统

#### 主色

- Pine 900: `#17352C`
- Pine 700: `#285443`
- Pine 500: `#3D7A61`

用途：

- 主按钮
- 关键操作
- 品牌识别

#### 辅助色

- Lake 500: `#3A7CA5`
- Sand 500: `#C59D5F`
- Clay 500: `#9B6B4F`

用途：

- 筛选标签
- 数据高亮
- 分类区分

#### 中性色

- Ink 950: `#111714`
- Ink 800: `#27312D`
- Stone 600: `#64706A`
- Stone 300: `#D7DDD9`
- Mist 100: `#F4F6F3`
- White: `#FFFFFF`

#### 功能色

- Success: `#2D8A57`
- Warning: `#C88519`
- Error: `#BE4D3F`
- Info: `#3A7CA5`

### 4.4 渐变与背景

- Hero 渐变：`#17352C -> #3D7A61`
- 地图遮罩渐变：`rgba(17,23,20,0.02) -> rgba(17,23,20,0.16)`
- 欢迎页背景：地形线浅纹 + 大面积自然摄影图遮罩

### 4.5 圆角与阴影

为了避免默认 MD 风格，采用偏克制的圆角和更真实的层级。

- 小组件圆角：10
- 卡片圆角：16
- 底部抽屉圆角：24
- 全屏媒体圆角：0

阴影建议：

- 卡片阴影：Y 8 / Blur 24 / Opacity 0.08
- 浮动按钮阴影：Y 12 / Blur 28 / Opacity 0.16
- 底部工具栏阴影：Y -4 / Blur 18 / Opacity 0.06

## 5. Typography

### 5.1 字体建议

统一风格建议：

- 中文：思源黑体 / Noto Sans SC
- 英文与数字：Avenir Next / Inter Tight 风格参考

说明：

- 中文字体保持清晰稳重
- 英文数字在海拔、里程、时长等信息展示中更具工具感

### 5.2 字号层级

- Display: 32 / 40 / Semibold
- H1: 28 / 34 / Semibold
- H2: 24 / 30 / Semibold
- H3: 20 / 26 / Medium
- Title: 18 / 24 / Medium
- Body: 16 / 24 / Regular
- Body Small: 14 / 20 / Regular
- Caption: 12 / 16 / Medium
- Data Large: 24 / 28 / Semibold

### 5.3 文案语气

- 清晰、直接、可信
- 重要提示避免情绪化表达
- 户外安全类内容需明确，不使用模糊措辞

## 6. 栅格与间距

### 6.1 设备基准

- iPhone 参考宽度：390pt
- Android 参考宽度：412dp

### 6.2 边距系统

- 页面左右边距：20
- 内容区主间距：16
- 卡片内边距：16
- 紧密信息间距：8
- 模块区块间距：24
- 大区块间距：32

### 6.3 触控规范

- 最小点击区域：44x44
- 列表项高度：56 以上
- 主按钮高度：52
- 小按钮高度：40

## 7. 通用组件规范

### 7.1 按钮

#### Primary Button

- 背景：Pine 700
- 文字：White
- 高度：52
- 圆角：14
- 用途：登录、开始记录、发布路线

#### Secondary Button

- 背景：Pine 700 8% 透明
- 描边：Pine 500 16%
- 文字：Pine 700

#### Tertiary Button

- 无底，文字强调
- 常用于“稍后再说”“跳过”“查看全部”

#### Danger Button

- 背景：Error
- 用途：删除草稿、放弃记录

### 7.2 输入框

风格：

- 非 Material 浮动标签
- 使用固定顶部标题 + 输入区域组合
- 输入框背景使用 `Mist 100`
- 边框使用低对比描边

状态：

- 默认
- 聚焦
- 错误
- 禁用
- 已填充

### 7.3 标签 Chips

类型：

- 难度标签
- 场景标签
- 季节标签
- 点位类型标签

视觉：

- 胶囊型但不使用典型 MD 圆角 999 风格
- 建议圆角 12，高度 32

### 7.4 路线卡片

字段：

- 封面图
- 路线标题
- 地区
- 距离 / 时长 / 爬升
- 难度等级
- 作者信息
- 收藏数或浏览量

卡片变体：

- 大图推荐卡
- 双列网格卡
- 列表紧凑卡
- 地图联动卡

### 7.5 点位卡片

字段：

- 点位类型图标
- 点位名称
- 当前记录时间
- 文本描述
- 图片或视频缩略图

### 7.6 地图浮层

地图页面统一使用分层浮卡：

- 顶部搜索/返回区
- 底部路线摘要卡
- 点位详情半屏抽屉

### 7.7 媒体组件

- 图片 4:3 优先
- 视频封面显示时长标记
- 支持左右滑动浏览媒体组
- 全屏浏览支持缩放与下拉关闭

## 8. 图标与插画规范

### 8.1 图标风格

- 线性为主，局部实心强调
- 线条粗细保持统一
- 不使用 Google 默认 Material Symbols 风格作为主体系
- iOS 可映射 SF Symbols，Android 使用自定义同风格图标库

建议图标语义：

- 轨迹
- 山峰
- 地图针
- 观景点
- 补给点
- 危险
- 视频
- 照片
- 收藏

### 8.2 插画风格

- 仅用于空状态、欢迎页、结果为空页
- 使用地形、山脊、路径线、营地元素
- 控制信息密度，避免儿童化

## 9. 动效规范

### 9.1 运动原则

- 所有动效服务于空间关系和任务反馈
- 记录过程优先快速反馈，不做花哨过渡

### 9.2 建议动效

- 卡片点击：110ms 轻微缩放
- 页面切换：220ms 横向推进或淡入
- 底部抽屉展开：260ms 上滑
- 地图与列表切换：交叉淡入 + 位移
- 成功发布：微弹性勾选动画

### 9.3 记录态特殊反馈

- 轨迹记录中，顶部状态点进行低频脉冲
- 新增点位成功后，地图 marker 快速扩散反馈
- 拍摄完成后，缩略图滑入底部点位栏

## 10. 无障碍与体验底线

- 文本对比度不少于 4.5:1
- 关键按钮最小 44x44
- 地图相关关键操作提供文字辅助，不依赖图标单独表达
- 颜色不作为唯一信息区分方式
- 录制中、上传中、失败重试状态必须明确
- 支持系统字体放大时关键内容不截断

## 11. 核心流程设计

### 11.1 浏览者流程

1. 打开 App
2. 浏览发现内容或直接搜索
3. 查看路线详情
4. 切换地图查看轨迹与点位
5. 浏览图片、视频、评论
6. 收藏或关注作者

### 11.2 分享者流程

1. 登录
2. 进入记录页
3. 配置起始信息
4. 开始记录轨迹
5. 途中添加点位并拍照/录像
6. 结束记录
7. 编辑路线信息
8. 发布路线

### 11.3 新用户 onboarding

1. 欢迎页了解产品价值
2. 注册
3. 选择兴趣偏好和常去区域
4. 进入发现首页

## 12. 页面详细设计

### 12.1 欢迎页

目标：

- 建立品牌印象
- 快速说明核心价值
- 引导登录和注册

布局：

- 顶部为全宽自然场景大图，叠加地形线纹理
- 中部为品牌名、Slogan、两行价值说明
- 底部主按钮“开始探索”
- 次按钮“已有账号，去登录”

Slogan 建议：

“记录你的每一步，也分享给下一位出发的人。”

### 12.2 登录页

元素：

- 顶部返回
- 标题“欢迎回来”
- 手机号 / 邮箱输入
- 密码输入
- 忘记密码
- 登录按钮
- 第三方登录区域
- 底部注册链接

交互：

- 输入框获得焦点时页面上移，保证键盘不遮挡
- 主按钮在必填项合法前保持禁用
- 登录失败时在输入区域下方提示具体原因

### 12.3 注册页

字段：

- 昵称
- 手机号或邮箱
- 密码
- 确认密码
- 勾选协议

可选：

- 邀请码

说明：

- 协议区域简洁，不影响注册主任务
- 首次注册成功后引导进入完善资料

### 12.4 完善资料页

字段：

- 头像
- 昵称
- 性别可选
- 常驻城市
- 徒步经验等级
- 感兴趣主题

目的：

- 为首页推荐和搜索默认筛选提供基础数据

### 12.5 发现首页

结构：

- 顶部品牌标题 + 消息入口 + 快速天气入口
- 搜索条
- 推荐 Banner
- 今日热门路线
- 周边可徒步
- 最新发布
- 精选作者

信息表现：

- 顶部区域背景有弱地形纹理
- 热门路线使用横滑大卡片
- 最新发布使用双列卡片
- 作者区使用横向头像列表

交互：

- 下拉刷新
- 卡片点击进入详情
- 搜索条点击进入搜索页

### 12.6 搜索页

默认状态：

- 搜索框
- 最近搜索
- 热门关键词
- 热门城市 / 山系专题

结果页模式：

- 顶部搜索框固定
- 下方 Tab：路线 / 点位 / 作者
- 结果支持列表模式和地图模式切换

筛选项：

- 距离
- 用时
- 难度
- 海拔爬升
- 适合季节
- 是否有补给
- 是否适合新手

### 12.7 地图结果模式

布局：

- 全屏地图
- 顶部返回与搜索框
- 底部结果卡片横向滑动

交互：

- 滑动底部卡片时地图自动高亮对应路线
- 点击 marker 弹出路线摘要
- 可快速切回列表

### 12.8 路线详情页

这是全产品最核心的消费页。

页面结构：

- 顶部封面图 / 视频封面
- 渐隐导航栏
- 路线标题
- 地区与发布时间
- 关键数据概览
- 作者信息
- 简介
- 地图轨迹预览
- 关键点位时间线
- 图片/视频集
- 评论区
- 底部固定操作栏

关键数据概览字段：

- 总里程
- 总时长
- 累计爬升
- 难度等级
- 最佳季节

底部固定操作：

- 收藏
- 分享
- 开始参考

“开始参考”含义：

- 收藏并进入地图查看模式
- 或准备下载离线资源

### 12.9 点位详情页

触发方式：

- 从路线详情中的时间线进入
- 从地图 marker 点击进入

内容结构：

- 点位名称
- 点位类型标签
- 时间戳
- 海拔信息
- 描述文字
- 图片/视频列表
- 与路线主轨迹的位置关系示意

点位类型：

- 起点
- 观景点
- 岔路口
- 危险提醒
- 补给点
- 休息点
- 终点

### 12.10 评论页

内容：

- 评论列表
- 发布评论输入框
- 点赞评论
- 回复楼层

规范：

- 评论输入框固定底部
- 支持插入位置标签，但默认文本优先

### 12.11 记录准备页

目标：

- 在真正开始记录前完成必要设置

字段：

- 路线名称
- 起点位置
- 计划公开范围
- 是否同步记录海拔
- 是否启用离线提醒

主操作：

- 开始记录

次操作：

- 保存为草稿

### 12.12 轨迹记录中页面

这是分享者端最核心的生产页面。

布局建议：

- 上半区为地图
- 下半区为关键数据和快捷操作面板

顶部区：

- 返回
- 录制状态
- GPS 精度
- 经过时长

地图区：

- 当前定位点
- 已绘制轨迹线
- 已添加点位 marker

底部操作区：

- 添加点位
- 拍照
- 录像
- 暂停
- 结束

数据区：

- 已走距离
- 当前海拔
- 累计爬升
- 已记录点位数

设计重点：

- 任何核心操作在强光下都应一眼可见
- 结束按钮与暂停按钮区分明确，避免误触
- 拍照与录像应支持一键进入，不要先强制填写表单

### 12.13 添加点位弹层

触发方式：

- 点击“添加点位”
- 拍照或录像后自动进入点位补充页

字段：

- 点位类型
- 标题
- 描述
- 自动带入经纬度与时间
- 添加照片
- 添加视频

交互：

- 默认优先展示“快速保存”
- 高级字段折叠在“补充更多信息”

### 12.14 拍照 / 录像页

设计原则：

- 与系统相机体验接近
- 强化点位记录语境

界面元素：

- 顶部关闭
- 当前模式切换：照片 / 视频
- 快门按钮
- 缩略图预览

## 13. 关键交互细节

### 13.1 路线卡片交互

- 点击卡片进入详情
- 长按卡片弹出快捷操作：收藏、分享、不感兴趣
- 横滑卡片时显示弱视差，不做夸张 3D

### 13.2 地图与内容联动

- 点路线时间线条目时地图自动居中到该点位
- 点地图 marker 时下方弹出点位摘要卡
- 地图拖动后出现“回到路线中心”按钮

### 13.3 记录过程降负担设计

- 点位可先拍后补
- 结束后集中编辑
- 网络不好时本地保存并延后上传
- 有未上传媒体时展示轻提示，不阻断继续记录

### 13.4 失败与异常反馈

- GPS 信号差：顶部黄条提示
- 媒体上传失败：在点位卡片内展示重试
- 发布失败：保留草稿并提示原因

## 14. iOS 与 Android 适配策略

### 14.1 共用内容

- 品牌色
- 字体层级
- 信息架构
- 组件样式
- 页面布局逻辑
- 动效节奏

### 14.2 iOS 差异

- 导航采用更轻的顶栏，支持大标题在发现页中出现
- 底部 Tab 高度结合安全区留白更充足
- 弹层优先使用 sheet 式呈现，转场更柔和
- 图标可映射 SF Symbols，但需保持与整体图标风格一致
- 开关、时间选择等系统组件可更贴近 iOS 原生

### 14.3 Android 差异

- 保持同品牌视觉，不使用默认 Material 组件外观
- 顶栏高度、返回热区适度增大
- 底部导航在视觉上更厚重一点以适应 Android 习惯
- 弹层支持更明确的拖拽指示器
- 文件、定位、相机等权限引导文案更明确

### 14.4 一致性边界

统一的是“产品体验与视觉气质”，不是强行像素级复制系统控件。

原则：

- 导航逻辑一致
- 视觉语言一致
- 组件形态高度一致
- 手势与返回行为遵循各平台习惯

## 18. 可用性风险与建议

### 风险 1

记录过程若步骤过多，分享者会放弃沉淀内容。

建议：

- 拍摄后先自动挂载点位
- 文本描述允许稍后补全

### 风险 2

路线详情如果地图信息不够直观，浏览者难以建立信任。

建议：

- 关键数据、轨迹图、点位时间线三者必须同屏有逻辑串联

### 风险 3

如果安全信息弱化，会损害产品可靠性。

建议：

- 难度、风险、季节、补给必须进入详情首屏或次首屏

## 20. 结论

TrailNote 的设计核心不在“做一个地图 App”，而在于建立一套适合户外路线沉淀和消费的内容体验系统。视觉上应体现自然探索感与可靠工具感，交互上则要让“记录”和“查看”都足够轻量、清晰、可信。iOS 与 Android 应共用同一套品牌与组件体系，但在导航、弹层、系统控件与手势反馈上保持各自平台的自然习惯。

这份文档可作为后续 Figma 页面设计、原型定义、组件沉淀和研发实现的直接基础。
